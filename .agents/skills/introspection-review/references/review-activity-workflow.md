# Review activity workflow

## Objective

Explain dashboard review activity using accepted manual-review facts only.

## Required actions

1. Exclude capability probes from review-activity counts.
2. Count only accepted classification or proposal review facts.
3. Report a factual zero only when the current aggregate is present and zero.
4. Report unavailable when no current aggregate is present.
5. Report not applicable when the metric does not apply to the selected operation.
6. State the resulting dashboard implication without estimating missing activity.

## Done when

- Review activity distinguishes zero, unavailable, and not applicable.
- The dashboard implication is based on accepted manual-review facts only.
