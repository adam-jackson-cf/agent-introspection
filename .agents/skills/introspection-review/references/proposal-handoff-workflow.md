# Proposal handoff workflow

## Objective

Separate classification evidence from any later proposal or repository decision.

## Required actions

1. Treat classifications as review facts only.
2. Do not alter findings, proposals, or repository files during classification.
3. State the next decision that requires an explicit separate user request.
4. Preserve accepted classifications for the authorised import path only.
5. Confirm that no repository mutation occurred.

## Done when

- The next decision is explicit.
- Classification has not changed findings, proposals, or repository files.
